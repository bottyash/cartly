## Blockers Faced

**Blocker:** Several PRD assumptions (A1 peak surge multiplier, A3 cost ceiling, A4 latency target,
A5 hard-escalation trigger list) are logged as unconfirmed client assumptions with no real Cartly
stakeholder to validate them against, since the client is fictional for this engagement.

**What we tried before flagging it:**
- Benchmarked against comparable public deployments (Intercom Fin, Zendesk AI, Klarna's 2-minute
  resolution figure) to ground the assumptions in real industry numbers rather than guesses
- Cross-checked the assumptions against each other for internal consistency (e.g. cost ceiling vs.
  model routing tier choices)
- Documented every assumption with a numbered reference (A1–A10) and an explicit "risk if wrong"
  so the team can move forward without blocking on an answer that doesn't exist

**Resolution path:** Proceeding with the stated assumptions as working baselines for Sprint 1
design work; each is flagged for revisit if a future stage (or faculty/mentor feedback standing in
for the "client") suggests a correction.
